# Architecture Overview

## Core Rule
AI generates JSON. Never HTML.
Frontend renders HTML from JSON via templates.

## Flow
```
Order input
  → OrderParserService      (AI → ParsedOrder JSON)
  → LandingGeneratorService (AI → LandingPageModel JSON)
  → ReplyGeneratorService   (AI → ReplyVariant[] with slug link)
  → DB stores everything
  → /r/{slug} renders landing from JSON
  → DialogueCopilotService  (AI → DialogueSuggestion JSON)
```

## Module Map
| Module     | Backend service             | Router prefix        |
|------------|-----------------------------|----------------------|
| projects   | —                           | /projects            |
| orders     | OrderParserService          | /orders              |
| replies    | ReplyGeneratorService       | /projects/{id}/replies |
| landings   | LandingGeneratorService     | /projects/{id}/landing |
| dialogue   | DialogueCopilotService      | /projects/{id}/dialogue |
| analytics  | —                           | /events              |

## Landing Rendering Pipeline
```
ParsedOrder
  → LandingGeneratorService
  → LandingPageModel (JSON in DB)
  → GET /r/{slug} (FastAPI returns JSON)
  → Next.js /r/[slug]/page.tsx renders via template
```

## Photo System
- Photos are NOT generated by AI
- AI selects photo_set_id based on event_type
- photo_set_id references a set of pre-uploaded photos in DB
- Frontend loads photos for that set_id

## Funnel Stages
new_lead → replied → opened → engaged → qualified → booked
